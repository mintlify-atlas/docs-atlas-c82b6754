## Overview

Patron Observer Java demonstrates a clean implementation of the Observer design pattern in Java, specifically designed for handling order events in an e-commerce system. This project showcases how to decouple event notification logic from business logic, enabling extensible and maintainable event-driven architectures.

## How It Works

The Observer pattern allows objects (observers) to watch and react to events from another object (subject). When an order is shipped, the system automatically notifies all registered observers:

1. **Email Observer** - Sends confirmation emails to customers
2. **Log Observer** - Records audit trail entries
3. **Stats Observer** - Updates shipment metrics

<Info>
The core principle is that `OrderService` doesn't need to know about email, logging, or statistics — it simply notifies the subject when an order ships, and all registered observers react automatically.
</Info>

## Use Cases

This pattern is ideal for:

- **Event-driven systems** where multiple actions must occur in response to a single event
- **Plugin architectures** where functionality needs to be extended without modifying core code
- **Audit systems** where multiple subsystems need to track state changes
- **Notification systems** where various channels (email, SMS, push) react to events
