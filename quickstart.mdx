## Overview

This guide walks you through using the Observer pattern to handle order shipment notifications. You'll learn how to set up a subject (event publisher), register multiple observers, and trigger notifications automatically.

<Steps>

<Step title="Create the Subject (Event Publisher)">

First, create an instance of `OrderEventPublisher`, which implements the `OrderSubject` interface. This object will manage all observers and notify them when events occur.

```java
OrderSubject publisher = new OrderEventPublisher();
```

The `OrderEventPublisher` maintains a list of observers and provides methods to add, remove, and notify them (OrderEventPublisher.java:8).

</Step>

<Step title="Create Observer Instances">

Create instances of each observer that will react to order shipment events. This implementation includes three concrete observers:

```java
OrderObserver emailObserver = new EmailOrderObserver();
OrderObserver logObserver = new LogOrderObserver();
OrderObserver statsObserver = new StatsOrderObserver();
```

Each observer implements the `OrderObserver` interface and defines its own `onOrderShipped()` behavior:
- **EmailOrderObserver**: Sends notification emails to customers
- **LogOrderObserver**: Logs order status changes
- **StatsOrderObserver**: Tracks shipment statistics

</Step>

<Step title="Subscribe Observers to the Subject">

Register all observers with the subject so they'll be notified when events occur:

```java
publisher.addObserver(emailObserver);
publisher.addObserver(logObserver);
publisher.addObserver(statsObserver);
```

The order of subscription determines the notification order. Observers are stored in an ArrayList and notified sequentially (OrderEventPublisher.java:22-24).

</Step>

<Step title="Create the Order Service">

Initialize the `OrderService` with the subject. The service uses dependency injection to receive the subject:

```java
OrderService orderService = new OrderService(publisher);
```

This decouples the service from specific observer implementations, adhering to the Dependency Inversion Principle (OrderService.java:7-9).

</Step>

<Step title="Create and Ship an Order">

Create a new order and trigger the shipment process:

```java
Order order = new Order("ORD-001");
orderService.shipOrder(order);
```

When `shipOrder()` is called (OrderService.java:11-19):
1. The order status is updated to "ENVIADO"
2. A confirmation message is printed
3. The subject's `notifyOrderShipped()` method is invoked
4. All registered observers are notified in sequence
5. Each observer executes its `onOrderShipped()` logic

</Step>

</Steps>

## Complete Example

Here's the full working example from `AppMain.java`:

```java
package patron_observer.marzo_2026_2;

public class AppMain {

    public static void main(String[] args) {

        // 1) Crear el Subject (publisher de eventos)
        OrderSubject publisher = new OrderEventPublisher();

        // 2) Crear observers
        OrderObserver emailObserver = new EmailOrderObserver();
        OrderObserver logObserver = new LogOrderObserver();
        OrderObserver statsObserver = new StatsOrderObserver();

        // 3) Suscribir observers al subject
        publisher.addObserver(emailObserver);
        publisher.addObserver(logObserver);
        publisher.addObserver(statsObserver);

        // 4) Crear el servicio usando el subject
        OrderService orderService = new OrderService(publisher);

        // 5) Crear un pedido y enviarlo
        Order order = new Order("ORD-001");
        orderService.shipOrder(order); // esta función llama a cada observer

    }
}
```

## Expected Output

When you run the application, you'll see the following console output:

```
OrderService: Pedido ORD-001 marcado como ENVIADO.
[EMAIL] Pedido ORD-001 enviado. Mandando email al cliente.
[LOG] Pedido ORD-001 ha cambiado a estado: ENVIADO
[STATS] Contador de pedidos enviados: 1
```

<Note>
The OrderService prints its message first, followed by each observer's notification in the order they were subscribed.
</Note>

## What Happens Under the Hood

When `shipOrder()` is called:

1. **Business Logic Execution** (OrderService.java:13-14)
   - The order's status is changed from "CREADO" to "ENVIADO"
   - A confirmation message is printed to the console

2. **Event Notification** (OrderService.java:18)
   - The service calls `orderSubject.notifyOrderShipped(order)`
   - This delegates to `OrderEventPublisher.notifyOrderShipped()`

3. **Observer Iteration** (OrderEventPublisher.java:22-24)
   - The publisher iterates through its list of registered observers
   - Each observer's `onOrderShipped()` method is invoked with the order

4. **Observer Reactions**
   - **EmailOrderObserver**: Prints email notification message (EmailOrderObserver.java:7)
   - **LogOrderObserver**: Logs the order status change (LogOrderObserver.java:7)
   - **StatsOrderObserver**: Increments internal counter and displays statistics (StatsOrderObserver.java:9-10)

<Info>
The Observer pattern decouples the OrderService from specific notification implementations. You can add or remove observers without modifying the service code.
</Info>
